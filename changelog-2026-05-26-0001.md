# Changelog 2026-05-26-0001

## 变更概述

修复 agent-dom-js 对 zTree（jQuery tree plugin）组件嵌套 `<li>` 元素的快照与点击支持。

## 背景

深澜认证计费系统使用 zTree 组件渲染用户组树。DOM 结构为 `<li>` 嵌套在 `<div class="ztree">` 中，每层 `<li>` 带 `tabindex="0"`。agent-dom-js 存在三个问题：

1. **父级 `<li>` 名称包含所有后代文本**：`getAccessibleName()` 回退到 `element.textContent`，递归拼接所有嵌套 `<li>` 的文本
2. **`<li>` ref 点击不触发 zTree 选择**：zTree jQuery 事件委托仅绑定 `a[treenode_a]`，不响应 `<li>` 直接点击
3. **快照中看不到选中状态**：zTree 用 `curSelectedNode` class 标记选中，但快照捕获的是 `<li>` 而非 `<a>`

## Review-0001

### 审查结论

本次 zTree 适配方向正确，但仍有非阻塞整改项。

1. `describeElement` 中 `cloneNode(true)` 用法本身正确，可排除嵌套 `ul/ol` 文本；与 `dom-utils.ts` 的 `<li>` 逻辑有重复。
2. `normalizeClickTarget` 条件偏宽，仅判断 `.ztree` 容器和 `<li>`，建议增加 zTree 节点特征约束（`treenode` 属性）。
3. `getAccessibleName()` 对所有 `<li>` 生效，会改变非 zTree 交互列表项的名称语义，需确认通用性。
4. 选中检测用 `/selected|current|active/` 正则可能误伤 `inactive`/`unselected`，建议精确 token 匹配。
5. 测试缺少 `createAgentDom()` 默认适配器集成测试和边界场景。

## Review-0002

### 审查结论

通过。

Review-0001 中要求的主要整改项已正确实施：

- `normalizeClickTarget` 已收紧到 `.ztree` 内的 `<li[treenode]>`，并仅重定向到直接子级 `a[treenode_a]`。

- `a` 变量命名已统一为 `anchor`。
- 已补充无 `treenode`、无 `a[treenode_a]`、非 zTree、非 zTree 嵌套 `<li>`、默认 `createAgentDom()` 集成 snapshot/click 等测试。
- `ztreeAdapter` 已注册到默认内置适配器列表。

### 仍保留事项

- `src/frameworks/ztree.ts` 与 `src/dom-utils.ts` 仍各自实现了一份 `<li>` 排除嵌套 `ul/ol` 文本的逻辑，属于轻微重复；当前不影响行为，可后续重构为共享 helper。
- `dom-utils.ts` 对全局 `<li>` 的名称修正确认合理：父级列表项不应默认包含嵌套列表文本。
