Agent DOM - zTree 嵌套 li 测试

模拟 zTree 组件 DOM 结构：<li> 嵌套在 <div class="ztree"> 中， 每层 <li> 有 tabindex="0"。 验证 agent-dom 对嵌套列表元素的快照与点击行为。

问题复现：模拟真实 zTree（仅响应 <a> 点击）

真实 zTree 通过 jQuery 事件委托 仅监听 <a> 元素的点击， 不响应 <li> 的直接点击。 因此 agent-dom 快照中 <li> 有 ref 但点击无效。

  • /(ID:1)
    • 用户组A(ID:2)
    • 用户组B(ID:3)
    • 二级组(ID:4)
      • 子用户组C(ID:5)
      • 子用户组D(ID:6)
    • 尚未选择任何节点

    根因分析

    问题 1：父级 <li> 名称包含所有后代文本

    getAccessibleName() 最终回退到 element.textContent， 这会递归获取所有后代文本。父级 <li> 的 name 变成： 

    /(ID:1) 用户组A(ID:2) 用户组B(ID:3) 二级组(ID:4) 子用户组C(ID:5) 子用户组D(ID:6)

    导致 AI 无法区分父节点名称 vs 子节点名称。

    问题 2：<li> 点击不触发 zTree 选择

    快照中的 <li> 匹配 [tabindex] 选择器，获得 ref。 但 zTree 的 jQuery 事件委托仅绑定在 <a[treenode_a] 上： 

    // zTree 内部实现等价于
$('.ztree').on('click', 'a[treenode_a]', function(e) {
  // 只处理 <a> 上的点击
});

    当 agent-dom 点击 <li> 时，事件 target 是 <li>， 不匹配 a[treenode_a] 选择器，zTree 不接受此事件。

    右侧面板中「快速测试」按钮同时提供了 li-ref 点击（无效）和 a-selector 点击（有效），可以对比验证。
