# Changelog: Highlight Feature

**Date:** 2026-06-10
**Branch:** feature/highlight

## Summary

新增 `AgentDom.highlight(target, options?)` 方法，用于以 RGB 跑马灯边框突出显示目标元素，并用四块半透明遮罩覆盖页面其他区域。

## Changes

### New Files

- `src/highlight.ts` — 高亮 DOM 生命周期管理模块
  - `highlightElement(element, options?)` — 创建四块遮罩 + RGB 动画边框
  - `HighlightHandle` — 返回 cleanup 句柄
  - 支持 `maskZIndex` 配置（默认 10）
  - 自动 `scrollIntoView` 到目标元素
  - 点击遮罩或目标元素关闭高亮（保留目标原点击行为）

- `test/highlight.test.ts` — 7 个单元测试
  - 默认/自定义 z-index
  - 遮罩点击关闭
  - 目标点击关闭并保留原行为
  - 非 HTMLElement / 不可见元素错误处理
  - DOM 异常时清理残留节点

### Modified Files

- `src/types.ts`
  - 新增 `HighlightOptions` 接口
  - `AgentDom` 接口扩展 `highlight` 方法

- `src/agent-dom.ts`
  - 导入 `highlightElement` 和 `HighlightHandle`
  - 闭包内维护 `activeHighlight` 状态，实现重复调用替换旧高亮

  - 新增 4 个公共 API 测试用例

## Key Design Decisions

1. **四块遮罩方案** — 避免全屏遮罩盖住目标元素，保证目标可见且可点击
2. **readOnly 下允许执行** — 高亮是视觉提示，不触发业务交互
3. **替换而非叠加** — 同一实例只维护一个高亮，重复调用先 cleanup 旧高亮
4. **捕获阶段监听目标点击** — 只做 cleanup，不阻止事件传播，保留原点击行为

### 优点

- API 形状、默认 `maskZIndex`、readOnly 放行、policy 复用与重复调用替换逻辑整体符合规格。
- 高亮生命周期被独立封装在 `src/highlight.ts`，没有继续污染动作模块，结构清晰。
- DOM 异常路径会返回 `ACTION_FAILED` 并执行 cleanup，目标点击也未阻止原事件行为。
- 测试覆盖了主要成功路径、错误路径、遮罩点击关闭、目标点击关闭与公共 API 接入。

### 问题

#### Critical (必须修复)

- [ ] 无。

#### Important (建议修复)

- [ ] 跨 document/window 的 HTMLElement 会被误判为不支持元素；`element instanceof HTMLElement` 使用当前全局构造器，若 `createAgentDom({ root })` 指向 iframe 或其他 realm，合法 HTML 元素会失败，建议基于 `element.ownerDocument.defaultView?.HTMLElement` 判断，并同步处理可见性依赖。`src/highlight.ts:18`
- [ ] 用户点击目标或遮罩后，`highlightElement` 已 cleanup，但 `createAgentDom` 闭包里的 `activeHighlight` 仍保留 handle，长生命周期实例会继续引用目标元素和已移除节点，直到下一次 highlight 或实例释放。建议让 cleanup 回调同步清空 `activeHighlight`，或封装一层 idempotent cleanup 管理状态。`src/agent-dom.ts:73`
- [ ] 遮罩尺寸未按 viewport 上下限夹紧，超宽/超高或部分越界目标会生成 `calc(100vw - Npx)` / `calc(100vh - Npx)` 的负值风险，可能导致右侧或底部遮罩失效。建议用 `innerWidth`/`innerHeight` 将 `left/right/top/bottom` clamp 到 `[0, viewport]` 后再计算四块遮罩。`src/highlight.ts:77`

#### Minor (可选修复)

- [ ] 测试未验证 `scrollIntoView` 被调用，也未断言四块遮罩的几何区域是否真的围绕目标，当前主要验证节点数量与 z-index，容易漏掉布局计算回归。`test/highlight.test.ts:35`
- [ ] 测试设计文档要求 `test/highlight.test.ts` 覆盖重复调用替换旧高亮，但实际仅在公共 API 测试覆盖；建议补一个 handle cleanup 幂等或直接重复调用行为测试，明确模块级契约。`test/highlight.test.ts:34`

### 整改建议

优先修复跨 realm 判断和 `activeHighlight` 状态释放；随后将遮罩矩形计算抽成可单测的纯函数，基于 viewport clamp 后生成四块区域，并补充越界/超大目标、`scrollIntoView`、遮罩几何断言等测试。整体实现无需重构，保持当前模块边界即可。
