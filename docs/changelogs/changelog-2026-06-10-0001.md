# Changelog: Highlight Feature

**Date:** 2026-06-10
**Branch:** feature/highlight
**Commit:** 7d1de77

## Summary

新增 `AgentDom.highlight(target, options?)` 方法，用于以 RGB 跑马灯边框突出显示目标元素，并用四块半透明遮罩覆盖页面其他区域。

## Changes

### New Files

- `src/highlight.ts` — 高亮 DOM 生命周期管理模块
  - `highlightElement(element, options?)` — 创建四块遮罩 + RGB 动画边框
  - `HighlightHandle` — 返回 cleanup 句柄
  - 支持 `maskZIndex` 配置（默认 10）
  - 自动 `scrollIntoView` 到目标元素
  - 点击遮罩或目标元素关闭高亮（保留目标原点击行为）

- `test/highlight.test.ts` — 7 个单元测试
  - 默认/自定义 z-index
  - 遮罩点击关闭
  - 目标点击关闭并保留原行为
  - 非 HTMLElement / 不可见元素错误处理
  - DOM 异常时清理残留节点

### Modified Files

- `src/types.ts`
  - 新增 `HighlightOptions` 接口
  - `AgentDom` 接口扩展 `highlight` 方法

## Key Design Decisions

1. **四块遮罩方案** — 避免全屏遮罩盖住目标元素，保证目标可见且可点击
2. **readOnly 下允许执行** — 高亮是视觉提示，不触发业务交互
3. **替换而非叠加** — 同一实例只维护一个高亮，重复调用先 cleanup 旧高亮
4. **捕获阶段监听目标点击** — 只做 cleanup，不阻止事件传播，保留原点击行为
