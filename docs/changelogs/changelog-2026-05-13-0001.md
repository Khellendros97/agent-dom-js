### 架构

参见设计文档 `docs/plans/2026-05-13-agent-dom-js-design.md`。

模块层次：
```
createAgentDom (facade)
  ├─ RefRegistry     ← ref ↔ Element 映射
  ├─ resolveTarget   ← 目标解析 (ref + CSS selector + policy)
  ├─ createSnapshot  ← DOM 扫描 + 文本渲染
  ├─ ActionEngine    ← click/fill/focus 执行
  ├─ WaitEngine      ← 条件轮询
  └─ SafetyPolicy    ← deny/allow/readOnly
```

### API 契约

```ts
interface AgentDom {
  snapshot(): AgentDomResult<SnapshotResult>;
  click(target: string): Promise<AgentDomResult<ActionResult>>;
  fill(target: string, value: string): Promise<AgentDomResult<ActionResult>>;
  focus(target: string): Promise<AgentDomResult<ActionResult>>;
  getText(target?: string): AgentDomResult<string>;
  isVisible(target: string): AgentDomResult<boolean>;
  waitFor(target: string, options?: WaitOptions): Promise<AgentDomResult<ActionResult>>;
}

type AgentDomResult<T> =
  | { ok: true; data: T }
  | { ok: false; code: AgentDomErrorCode; error: string; detail?: unknown };
```

### 已知限制

- 快照是 DOM + ARIA 近似，不是真实 accessibility tree

## Review-0001

### 审查结果
需整改

### 整改事项
- [ ] 收窄 `src/index.ts` 公共导出，只暴露稳定 facade、公共类型和必要 result helper，避免把内部引擎作为 API 契约暴露。
- [ ] 为 `click` / `fill` / `focus` 增加 disabled、可见性和异常捕获校验，避免裸异常和误操作禁用控件。
- [ ] 将 `allowSelectors` / `denySelectors` 应用于 `snapshot()`，至少不要为被策略禁止的元素生成 ref。
- [ ] 补齐 React/Vue 真实 fixture 或浏览器级 smoke 测试，当前只验证了静态 HTML/jsdom。
- [ ] 增强快照信息：输入类型、placeholder、select options、link href、层级/上下文，提升 AI 可用性。
- [ ] 修正 `waitFor()` 对 `POLICY_BLOCKED` 等确定性错误的处理，不应全部吞掉后返回 `TIMEOUT`。
- [ ] 处理非法 policy selector、DOM action 异常和缺失 `InputEvent` 等浏览器兼容边界。

### 审查意见
整体架构方向正确，模块拆分与设计文档基本一致，MVP 代码量克制，没有明显过度抽象。`AgentDom facade + RefRegistry + Snapshot + Action + Policy` 的边界清晰，统一 `AgentDomResult` 也符合设计目标。

主要问题在公共 API 和安全边界。`src/index.ts:2` 起导出了 `clickElement`、`RefRegistry`、`createSnapshot`、`resolveTarget`、`waitForCondition` 等内部实现，这会过早固化内部模块契约，违背设计文档中 facade 隐藏细节的目标。建议 MVP 公共入口只保留 `createAgentDom`、公共类型，以及确有必要的 `success/failure`。

安全策略默认值中 `maskSensitiveValues` 默认开启是正确的，但 `snapshot()` 只传入了 masking 配置，未应用 `allowSelectors` / `denySelectors`，见 `src/agent-dom.ts:20` 和 `src/snapshot.ts:17`。这会导致被禁止区域仍出现在 AI 快照中并获得 ref。即使动作阶段会拦截，AI 已经看到敏感上下文，安全模型不完整。

动作引擎缺少关键校验。`src/actions.ts:5` 的 click 检查可见性但不检查 disabled；`src/actions.ts:27` 的 fill 不检查可见性和 disabled；`src/actions.ts:13`、`src/actions.ts:33`、`src/actions.ts:34` 周围也没有 try/catch，违反"不抛出裸异常给宿主"的错误处理原则。`DISABLED` 错误码已定义但实际未使用。

框架兼容性还不足。`src/actions.ts:48` 使用原生 value setter 是正确方向，但测试只覆盖事件派发，没有真实 React/Vue 受控组件验证。`test/fixtures.test.ts:7` 只读取静态 HTML，不能证明 React/Vue fixture 可用。

快照对 AI 已具备基础可用性，但信息偏少。`src/snapshot.ts:77` 只输出 role/name/ref/value/checked/disabled，缺少 input type、placeholder、select options、link href、父级语义上下文等。MVP 可接受，但作为 AI 操作层会影响目标选择准确性。

结论：实现已达到可演示 MVP，但不建议作为稳定公共 API 发布。完成以上整改后，再补一次 typecheck/build/browser fixture 验证即可进入通过状态。
